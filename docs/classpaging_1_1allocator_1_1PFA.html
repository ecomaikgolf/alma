Detailed Description

Page Frame Allocator class

Mantains a bitset (size of the total ammount of pages) that indicates free/used pages. You can lock a page/s or free it, you can also request a free page.

@info Requesting a free page can be improved with a free list (now it iterates from the start searching for a free page). For now it's out of scope, anyway can be easily implemented I think

paging::allocator::PFA::PFA ( const uefi::memory::map *  map)

Page Frame Allocator constructor

Get the largest segment and reserves it for the PFA class (to ensure it won't run out of memory). This is quite hacky and probably in real industry this is done in other (better) way.

Then "create" the bitset (set the buffer and size). Size is calculated from UEFIMMap::get_memsize(map) (total UEFI memory). Then we divide it into page size to match 1 bit to 1 page.

Then we zero out the bitset and lock the bitset pages.

Finally we go through all the UEFI memory map to reserve unusable pages
